### B-5 — 일봉 정렬 방향 불일치 (목표가 오류 가능)

**파일**: `autotrader/market_data.py`, `autotrader/market_filter.py`

**문제**

`get_prev_day_bar()`는 KIS API가 "최신 우선(내림차순)"으로 반환한다고 가정해 첫 번째 오늘과 다른 봉을 전일봉으로 반환한다. 반면 `market_filter.py:54`는 `sorted(bars, key=lambda b: b.date)`로 오름차순 재정렬한다. 만약 KIS API가 오름차순으로 반환하면 `get_prev_day_bar()`는 가장 오래된 봉을 전일봉으로 반환하여 목표가가 완전히 틀려진다.

**수정 방법**

1. `kis_broker.py`의 `get_daily_bars()` 반환 시 정렬 방향을 명시적으로 고정한다.
2. `get_prev_day_bar()`와 `market_filter.py` 모두 동일한 방향(내림차순)을 전제하도록 통일한다.

```python
# kis_broker.py get_daily_bars() 반환 직전 추가
bars.sort(key=lambda b: b.date, reverse=True)  # 최신 우선으로 정규화
return bars
```

```python
# market_filter.py:54 — 내림차순 정렬로 변경
sorted_bars = sorted(bars, key=lambda b: b.date, reverse=True)
closes = [b.close for b in sorted_bars if b.close > 0]
ma = sum(closes[:usable]) / usable   # 최신 N개의 평균
current = closes[0]                  # 가장 최신 종가
```

### R-3 — 봇 프로세스에 WAL 모드 미설정

**파일**: `autotrader/state.py:80~90`

**문제**

FastAPI(`main.py:63`)와 텔레그램 봇(`telegram_control.py:78`)은 연결마다 `PRAGMA journal_mode=WAL`을 실행하는데, 메인 봇의 `StateManager._conn()`에는 없다. 봇이 먼저 DB를 생성하면 기본 DELETE 모드로 초기화되고, 이후 FastAPI가 WAL로 전환하더라도 세 프로세스 간 동시 접근 시 쓰기 잠금 충돌이 발생할 수 있다.

**수정 방법**

`state.py`의 `_conn()` 또는 `_init_db()`에 WAL 설정을 추가한다.

```python
# state.py _init_db() 첫 줄 추가
def _init_db(self) -> None:
    with self._conn() as cx:
        cx.execute("PRAGMA journal_mode=WAL")   # ← 추가
        cx.executescript("""
            CREATE TABLE IF NOT EXISTS ...
        """)
```

### U-3 — UI_SECRET_KEY 클라이언트 번들 노출

**파일**: `ui/dashboard/lib/api.ts:2`

**문제**

```typescript
const SECRET = process.env.NEXT_PUBLIC_API_SECRET ?? "";
```

`NEXT_PUBLIC_` 접두사는 Next.js가 빌드 시 클라이언트 번들에 값을 포함시킨다는 의미다. 브라우저 개발자 도구 → 소스에서 `UI_SECRET_KEY` 값이 평문으로 보이며, 외부에서 복사해 `/kill` 등을 직접 호출할 수 있다.

**수정 방법**

제어 명령(`/kill`, `/resume`, `/watchlist` 등)은 Next.js API Routes를 통해 서버 사이드에서 FastAPI를 호출하는 방식으로 변경한다.

```
브라우저 → Next.js API Route (서버, SECRET 보유) → FastAPI
```

단기 임시 조치로는 인증이 필요 없는 내부망 환경이라면 `UI_SECRET_KEY`를 비워두는 방법이 있다.

### O-5 — `_token_cache.json`이 cwd 상대경로

**파일**: `autotrader/kis_broker.py:39`

**문제**

```python
_TOKEN_CACHE_FILE = Path("_token_cache.json")
```

봇, FastAPI, 텔레그램 봇이 서로 다른 작업 디렉토리에서 실행되면 각자 다른 토큰 캐시 파일을 사용하여 불필요한 토큰 재발급이 발생한다.

**수정 방법**

환경변수로 경로를 지정하거나 `STATE_DB`와 동일한 디렉토리를 사용한다.

```python
_TOKEN_CACHE_FILE = Path(os.getenv("TOKEN_CACHE_FILE", "_token_cache.json"))
```
