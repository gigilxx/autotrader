# 나만의 주식 자동매매 프로그램 — MVP 기획서 초안

> 단타/스윙 전용 · 한국투자증권(KIS) Open API 기반 · 바이브 코딩(Claude) 제작
> 버전 0.1 (초안) · 작성 목적: MVP 범위 합의 및 개발 출발점

---

## 0. 한 줄 요약

> **"좋은 매수 타이밍을 찾아주는 봇"이 아니라, "정해진 규칙을 감정 없이 지키고, 절대 크게 잃지 않게 막아주는 봇"을 먼저 만든다.**

---

## 1. 문제 정의 (Why)

| 항목 | 내용 |
|---|---|
| 현재 문제 | 주식 이해도가 낮고, 매수·매도 시점을 잘 못 잡아 투자 손실 비율이 높음 |
| 흔한 오해 | "타이밍을 못 잡으니 자동화로 타이밍을 대신 잡게 하자" |
| 진짜 원인(통계적) | 초보 손실의 대부분은 *진입 실패*가 아니라 **손절 실패 · 과도한 포지션 · 물타기 · 감정적 추격매수** = **리스크 관리 부재** |
| 따라서 MVP의 진짜 목표 | 매수 신호를 똑똑하게 만드는 것보다, **손절·포지션 크기·일일 손실 한도를 기계가 강제**하게 만드는 것 |

> 💡 핵심: 자동화는 판단을 대신해주지 않는다. 내가 짠 규칙을 더 빠르고 일관되게 실행할 뿐이다.
> → **검증 안 된 전략의 자동화 = 손실을 더 빠르게 내는 기계.** 그래서 "검증" 단계가 이 프로젝트의 절반이다.

---

## 2. 설계 철학: "진입 알고리즘"이 아니라 "규율 엔진"

봇의 구성 요소를 우선순위 순으로:

1. 🛡️ **리스크 관리 레이어 (최우선)** — 손절, 포지션 한도, 일일 최대손실, 킬스위치
2. 📏 **단순·검증 가능한 진입/청산 규칙** — 복잡한 AI 예측 ❌ / 백테스트 가능한 명확한 규칙 ✅
3. 🤖 **실행(주문) 레이어** — API로 정확하고 안정적으로 주문 실행
4. 📊 **모니터링/로깅** — 무슨 일이 일어났는지 전부 기록·알림

> 진입 신호가 50점짜리여도 리스크 관리가 100점이면 살아남는다.
> 진입 신호가 100점이어도 리스크 관리가 0점이면 한 방에 깡통이다.

---

## 3. MVP 범위 (Scope)

### ✅ MVP에 포함 (할 것)
- KIS **모의투자** 계좌 연동 (실전 아님)
- 관심종목 1~5개에 대한 **단순 규칙 기반** 매수/매도 신호 생성
- **손절(stop-loss) 자동 실행** (가장 중요)
- **포지션 사이징**: 1종목당 최대 투입 비율 제한
- **일일 최대 손실 한도** 도달 시 당일 매매 중단
- **킬스위치**: 수동/자동 전체 정지 버튼
- 모든 주문·신호·에러 **로깅** + 텔레그램/이메일 **알림**
- **백테스트 모듈**: 과거 데이터로 전략 성과 검증

### ❌ MVP에서 제외 (나중에)
- 실전(실제 돈) 매매 — *모의 성과 기준 통과 전까지 절대 ❌*
- 다종목 동시 스캔/스크리닝 (전체 시장 탐색)
- 머신러닝/딥러닝 가격 예측 모델
- 화려한 대시보드 UI (CLI/간단 로그로 시작)
- 해외주식, 선물/옵션, 레버리지

## 4. 기술 스택 & API 선택

### API: 한국투자증권 KIS Developers (권장)

| 기준 | KIS API | 비고 |
|---|---|---|
| 방식 | REST + WebSocket | Windows/32bit 종속 없음, Mac/Linux 가능 |
| 모의투자 | ✅ 별도 앱키/시크릿으로 지원 | **같은 코드로 모의↔실전 전환 가능** |
| 레퍼런스 | 공식 GitHub 샘플(백테스터·전략빌더 포함), 커뮤니티 활성 | 바이브 코딩 친화적 |
| AI 연동 | ChatGPT·Claude 연동 공식 지원/문서화 | Claude로 만들기 최적 |

> 미래에셋증권 API도 모의/실전 모두 제공하지만, 레퍼런스·커뮤니티가 훨씬 적어 초기 시행착오가 큼. **KIS로 시작 → 검증 후 필요 시 어댑터 패턴으로 교체** 권장.

### 추천 스택 (바이브 코딩 기준)
- **언어**: Python 3.11+ (자동매매 생태계·예제 압도적으로 풍부)
- **HTTP/WS**: `requests`, `websockets` (또는 커뮤니티 라이브러리 `python-kis`)
- **데이터/지표**: `pandas`, `numpy`, `pandas-ta`(기술지표)
- **백테스트**: `backtesting.py` 또는 KIS 공식 backtester
- **스케줄링**: `APScheduler` (장 시작/종료, 주기적 체크)
- **알림**: `python-telegram-bot` (텔레그램 봇)
- **설정/보안**: `.env` + `python-dotenv` (앱키 절대 코드에 하드코딩 ❌, Git 커밋 ❌)
- **저장**: SQLite (거래 로그) → 나중에 Supabase로 확장 가능

> 참고: 평소 쓰던 Next.js/TS 스택으로 모니터링 대시보드를 따로 붙이는 건 Phase 4 이후 선택. 매매 엔진 본체는 Python이 압도적으로 유리.

## 5. 시스템 아키텍처 (모듈 구성)

```
┌─────────────────────────────────────────────┐
│              스케줄러 (장중 루프)               │
└───────────────┬─────────────────────────────┘
                │
   ┌────────────▼────────────┐
   │   1. 시세 수집 모듈        │  ← KIS WebSocket/REST (현재가, 호가, 분봉)
   └────────────┬────────────┘
                │
   ┌────────────▼────────────┐
   │   2. 전략/신호 모듈        │  ← 단순 규칙으로 BUY/SELL/HOLD 신호 생성
   └────────────┬────────────┘
                │  (신호)
   ┌────────────▼────────────┐
   │   3. 🛡️ 리스크 게이트     │  ← ★핵심★ 신호를 "허가/차단"
   │   - 손절가 체크            │     포지션 한도 초과? 일일손실 한도? 킬스위치?
   │   - 포지션 크기 계산        │     → 통과한 주문만 4번으로
   └────────────┬────────────┘
                │  (검증된 주문)
   ┌────────────▼────────────┐
   │   4. 주문 실행 모듈        │  ← KIS 주문 API (매수/매도/취소)
   └────────────┬────────────┘
                │
   ┌────────────▼────────────┐
   │   5. 로깅 & 알림 모듈      │  ← 모든 이벤트 기록 + 텔레그램 푸시
   └─────────────────────────┘
```

> ★ 3번 리스크 게이트가 이 프로젝트의 심장. 어떤 신호도 여기를 통과하지 않으면 주문되지 않는다.

## 6. 매매 전략 (초보자용 출발점)

> ⚠️ 아래 전략들은 *추천*이 아니라 *백테스트로 검증할 후보*입니다. 어느 것도 수익을 보장하지 않습니다.
> MVP의 목적은 "이기는 전략 찾기"보다 **"규칙을 기계적으로 지키는 시스템 만들기"**임을 기억하세요.
> 단순하고 규칙이 명확한 전략부터 시작하는 이유: **백테스트가 가능하고, 왜 사고팔았는지 설명이 되기 때문.**

### 후보 전략 (택1로 시작 권장)

**A. 이동평균 교차 (스윙 적합, 가장 단순)**
- 매수: 단기 이평선(예: 5일)이 장기 이평선(예: 20일)을 위로 돌파
- 매도: 반대로 교차 OR 손절가 도달
- 장점: 이해 쉽고 백테스트 명확 / 단점: 횡보장에서 잦은 손실(휩쏘)

**B. 변동성 돌파 (단타 적합)**
- 당일 시가 + (전일 변동폭 × k) 가격을 돌파하면 매수, 종가 청산
- 단타/데이트레이딩에서 가장 많이 쓰이는 검증된 기본형

**C. RSI 과매도 반등 (스윙)**
- RSI 30 이하 진입 → 일정 회복 시 매도
- 단, 추세 하락장에서 위험 → 손절 필수

### ⛔ 모든 전략 공통 — 절대 규칙 (전략보다 우선)
1. **손절선 필수**: 매수 시점에 손절가를 함께 정한다. 예외 없음.
2. **익절선 설정**: 목표 수익 도달 시 자동 매도 (욕심 차단)
3. **물타기 금지**: 손실 종목 추가 매수 ❌ (코드 레벨에서 차단)
4. **1종목 집중 금지**: 단일 종목 최대 비중 제한

## 7. 🛡️ 리스크 관리 규칙 (이 문서의 핵심)

> 이 표의 숫자는 **예시**입니다. 본인 성향·자금에 맞게 조정하되, "한도를 둔다"는 원칙 자체는 타협 금지.

| 규칙 | 예시 값 | 목적 |
|---|---|---|
| 종목당 손절 | -2% ~ -3% | 한 번에 크게 잃지 않기 |
| 종목당 최대 투입 | 총자산의 10~20% | 한 종목 몰빵 방지 |
| 일일 최대 손실 | 총자산의 -3% | 도달 시 **당일 매매 자동 중단** ("오늘은 나쁜 날") |
| 동시 보유 종목 수 | 최대 3~5개 | 관리 가능 범위 유지 |
| 1일 최대 거래 횟수 | 예: 10회 | 과매매(오버트레이딩) 방지 |
| 킬스위치 | 수동 + 자동 | 이상 징후 시 전체 정지 |

### 킬스위치(전체 정지)가 작동해야 하는 상황
- 일일 손실 한도 도달
- API 연속 에러 / 시세 데이터 끊김
- 예상치 못한 큰 슬리피지(체결 가격 이상)
- 사용자가 수동으로 누름

## 8. 개발 단계 (Phase) — 단계별 게이트 통과제

> 각 Phase의 **완료 기준을 통과해야만** 다음으로 넘어간다. 특히 실전(돈) 진입 게이트는 엄격하게.

### Phase 0 — 환경 세팅 & 모의 연결 (1~2일)
- KIS Developers 가입, **모의투자 계좌 + 모의 앱키** 발급
- 인증(OAuth 토큰 발급) → 현재가 조회 1건 성공
- `.env` 보안 세팅, Git에 키 노출 안 되게 `.gitignore`
- ✅ 완료 기준: 모의계좌로 삼성전자 현재가 조회 + 1주 모의 매수 성공

### Phase 1 — 데이터 & 백테스트 (3~5일)
- 과거 분봉/일봉 데이터 수집
- 전략 A(또는 B) 백테스트 구현
- ✅ 완료 기준: 최근 6개월~1년 데이터로 전략 성과(승률, MDD, 손익비) 리포트 산출

### Phase 2 — 리스크 엔진 (가장 공들일 단계, 3~5일)
- 손절/익절/포지션 사이징/일일한도/킬스위치 구현
- ✅ 완료 기준: 일부러 손실 시나리오 넣어도 한도 넘으면 자동 차단되는지 테스트 통과

### Phase 3 — 모의 실시간 매매 (2~4주, ★가장 중요★)
- 전 모듈 결합, **모의계좌로 실시간 자동매매 가동**
- 로깅 + 텔레그램 알림
- ✅ **실전 진입 게이트(통과 못하면 절대 실전 ❌)**:
  - 최소 4주 이상 모의 운영
  - 치명적 버그 0건 (잘못된 주문/손절 미작동 0)
  - 리스크 규칙이 한 번도 안 뚫림
  - 결과를 본인이 **설명할 수 있음** (왜 이 거래를 했는지)

### Phase 4 — 실전 극소액 (신중하게)
- **잃어도 괜찮은 최소 금액**으로만 시작 (예: 수십만 원)
- 모의와 실전의 차이(슬리피지, 체결 지연, 수수료/세금) 확인
- 한동안 매일 모니터링, 문제 시 즉시 킬스위치

### Phase 5 — 점진적 개선
- 성과 분석 → 전략/파라미터 보수적 조정
- 필요 시 대시보드(Next.js) 추가

## 9. 안전장치 & 현실/법적 고려사항

- 🔐 **API 키 보안**: `.env`로만 관리, Git 커밋 금지, 유출 시 즉시 재발급
- 📉 **모의 ≠ 실전**: 모의는 슬리피지·미체결·세금이 다르게 잡힘. 모의 수익 ≠ 실전 수익
- 💸 **비용**: 거래세·수수료가 단타에선 수익을 크게 갉아먹음 → 백테스트에 **반드시 비용 반영**
- ⚖️ **본인 책임**: KIS 공식 안내도 "샘플 코드로 만든 프로그램의 손해는 책임지지 않음" 명시. 자동매매의 모든 결과는 본인 책임
- 🚦 **API 호출 제한**: 모의계좌는 호출 제한이 낮음 → 과도한 폴링 주의
- 🧠 **무인 방치 금지(초기)**: MVP 단계에선 봇을 완전 무인으로 돌리지 말고, 장중 모니터링 병행

## 10. 성공 기준 (MVP가 "성공"이라는 정의)

이 MVP의 성공은 **"돈을 벌었다"가 아니다.** 다음이면 성공:

1. ✅ 모의계좌에서 정해진 규칙대로 4주 이상 안정적으로 자동매매가 돌아감
2. ✅ 손절·일일한도·킬스위치가 실제로 작동함 (한 번도 안 뚫림)
3. ✅ 모든 거래가 로깅되고, 왜 그 거래를 했는지 설명 가능
4. ✅ 백테스트로 전략의 강점·약점을 객관적으로 알게 됨

> 이 4개를 만족하면, 실제 수익은 그 다음 문제. **"크게 잃지 않는 시스템"을 먼저 완성하는 것**이 MVP의 진짜 목표.
