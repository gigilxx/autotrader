# 엣지케이스 · 위험 · 가정 (솔직한 한계)

이 코드는 **커스텀 로직의 골격과 안전장치**를 검증된 형태로 제공하지만,
실전 무인 운용에 바로 쓰기엔 아래 항목들을 반드시 추가로 다뤄야 합니다.

## 가정 (Assumptions)
- 가격은 원(KRW) **정수**, 수량은 **정수**(소수 주식 없음).
- 단일 스레드 이벤트 루프 — 동시성 락(lock)은 없음.
- 브로커(KIS)는 **추상화**돼 있고, 실제 tr_id·엔드포인트·주문 파라미터는
  한투 공식 샘플에서 채워야 함. ⚠️ 공식 문서 확인 필요.
- **세율 0.20%·수수료 0.015%는 자리표시값** — 본인 계좌/현행 기준으로 확인.
- 신규 진입은 **지정가만** 허용(슬리피지 통제 우선). 정책상 가정이며 변경 가능.
- 일일 손실 추적은 **호출자가 청산 시 `register_realized_pnl()`을 호출**해야 갱신됨.
- 시간 판단은 **KST 로컬 시각**을 호출자가 정확히 넘긴다고 가정.

## 이 코드가 아직 다루지 않는 위험 (중요)
1. **부분 체결(partial fill)**: `send_order`가 bool만 반환. 실제로는 일부만
   체결될 수 있어 수량·잔고 회계가 어긋남 → 체결 수량 기반 처리 필요. ⚠️
2. **주문 거부·대기·취소**: 미체결/거부/정정 상태 처리 없음(성공 bool만).
3. **정확히 1회 전송(exactly-once)**: 멱등성 가드는 프로세스 내 중복은 막지만,
   전송 직후~`mark_sent` 사이 크래시 시 재전송으로 이중 주문 가능 →
   **잔고 대조(reconcile)를 백스톱으로** 사용. 완전 방탄 아님.
4. **강제청산 실패**: 마감 직전 청산 주문이 실패하면 못 보는 사이 **오버나이트**
   보유 위험 → 재시도 + 알림 에스컬레이션 필요(미구현).
5. **상태 비영속(no persistence)**: `trades_today`, 전송된 주문 id 등이 메모리에만
   있음 → **프로세스 재시작 시 소실**. 하루 종일 도는 무인 봇엔 디스크/DB 저장 필수. ⚠️
6. **미실현 손실 미반영**: 일일 한도는 **실현 손익만** 봄. 큰 평가손이 나도
   청산 전까진 정지 안 됨 → 평가손 기준 정지도 고려.
7. **시장 특수상황**: 상한가/하한가, VI(변동성완화장치), 거래정지 종목,
   동시호가/휴장일 캘린더 미처리 → 주문 거부·오작동 가능.
8. **유동성/호가 깊이**: 사이징이 종목 유동성을 보지 않음 → 저유동성 종목에서
   슬리피지·미체결 위험.
9. **토큰 갱신·호출 제한**: 브로커 레이어 책임 — 여기 미포함. ⚠️ KIS 문서 확인.
